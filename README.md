## 第4章｜Unixの本当の意義  
── in / process / out を失わないための設計

- 多くの作業は **in / process / out** の構造を持つ
- GUI操作では **process が操作として消費され、残らない**
- Unixは process を隠さず、再利用可能な形で残した
- pipe / filter は、処理を分解し連結するための仕組み
- 小さな試行錯誤を積み重ね、目的の out に近づくための思想

Unixの価値は速度や効率ではなく、  
**「考えた過程を失わない構造」を与えたこと**にある。
