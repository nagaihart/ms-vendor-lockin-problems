# ms-vendor-lockin-problems

教育機関・官公庁・組織のIT運用において発生している  
**ベンダーロックイン（特定製品・特定企業への依存）がもたらす構造的問題**を、  
専門知識のない人でも確認できる形で整理・公開する資料集です。

本リポジトリは、特定の企業や製品を批判することを目的としません。  
**「わからないまま導入・運用されてきた構造」**が、  
どのように現場・教育・将来世代へ影響しているのかを、  
事実ベースで可視化することを目的とします。

---

## この資料が対象とする読者

- 保護者・教育関係者
- フリースクール・オルタナティブ教育の運営者
- 官公庁・学校の意思決定に関わる人
- ITに詳しくないが、現状に違和感を持っている人

---

## ここで扱う問題意識

- Excel / Word による文書管理が、なぜ引き継げないのか
- レガシーシステムの改修が、なぜ高額で困難になるのか
- 「操作はできるが、仕組みはわからない」教育が何を生むのか
- その構造が、どのように次の世代へ再生産されているのか

これらは技術の良し悪しではなく、  
**理解・記録・再現が軽視されてきた運用構造の問題**です。

---

## 本資料が示したい一点

> 問題は「ITが難しいこと」ではない。  
> **「わからないものを、わからないまま使い続ける構造」**である。

そして、この構造は教育によって再生産されている。

---

## Scripter 構想との関係

本資料は、以下の教育的観点と接続されています。

- わからないことを前向きに扱う
- 「どこがわからないか」を特定できる
- 角度・道具・発想を変えて向き合える
- ときに休み、離れ、また拾い直せる

**解けることより、向き合い続ける姿勢**を育てるための基礎資料です。
